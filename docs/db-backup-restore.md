# 数据库备份与恢复说明（SQLite）

## 1. 数据文件位置
- 本地默认 SQLite 文件：`backend/reconciliation.db`

## 2. 手动备份
- 停止后端服务后，直接复制数据库文件并按日期命名，例如：
  - `reconciliation_20260407.db`
- 建议同时备份 `backend/.env`（如有）用于记录环境配置。

## 3. 手动恢复
- 停止后端服务。
- 将备份文件复制回 `backend/` 目录并重命名为 `reconciliation.db`。
- 重新启动后端服务验证数据是否正常。

## 4. 注意事项
- 恢复会覆盖当前数据，请先备份现有数据库文件。
- 若线上使用 PostgreSQL，请使用数据库平台提供的快照/导出能力，不能直接套用 SQLite 文件恢复。
